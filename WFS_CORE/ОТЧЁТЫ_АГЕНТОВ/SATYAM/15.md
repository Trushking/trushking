### 🔍 Почему код не индексируется?

Стандартные страницы на GitHub — это просто текст в репозитории. Чтобы поисковая система занесла код в свой индекс, необходимо «подсказать» ей, какой вес придавать каждому элементу. В противном случае алгоритмы просто не обращают внимания на «LR80TT32CE9X» в потоке других данных.

Решается это стандартными методами SEO: мета-тегами, картой сайта, микроразметкой и ручной отправкой ссылки на проверку.

---

### 🛠️ Как это можно исправить

#### 1. Превратить код в значимый элемент
Прямо сейчас код — это обычный текст. Его нужно преобразовать в значимый для системы элемент — например, в ссылку на файл.

**Рекомендуемый формат:**

```html
<a href="/путь_к_файлу.md" title="Идентификатор актива #WFS: LR80TT32CE9X">LR80TT32CE9X</a>
```
Это сразу дает поисковой системе больше контекста: код — это важный идентификатор, а не случайный набор символов.

#### 2. Добавить микроразметку (Schema.org/Product)
Самый эффективный способ — «подсказать» поисковой системе, что этот код является идентификатором ценного актива.

**Пример кода (можно вставить в HTML-шапку страницы):**

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org/",
  "@type": "Product",
  "name": "Актив #WFS: LR80TT32CE9X",
  "description": "Уникальный идентификатор тонкоматериального актива Церкви #WFS",
  "sku": "LR80TT32CE9X"
}
</script>
```
Это максимально конкретный способ объяснить роботу, что перед ним — не случайный набор символов, а важный идентификатор (SKU).

#### 3. «Показать» путь поисковым системам через sitemap.xml
Поскольку страницы #КЗ, вероятно, находятся в папке `WFS_CORE/ОПРЕДЕЛЕНИЯ/` (или другой), необходимо создать файл `sitemap.xml`, который перечислит все важные URL. Это стандартный протокол, который гарантирует, что робот не пропустит нужные страницы.

#### 4. Подключить Jekyll (если сайт еще не использует конструктор)
GitHub Pages отлично работает с Jekyll — системой, которая превращает Markdown-файлы в полноценные веб-страницы. Jekyll позволяет легко управлять мета-тегами, картой сайта и автоматически генерировать правильную HTML-разметку.

**Простой способ активировать Jekyll** — добавить в корень репозитория пустой файл `.nojekyll`.

#### 5. «Попросить» Google проиндексировать страницу вручную
Даже после всех настроек машине может потребоваться время, чтобы найти страницу. Ускорить процесс можно через инструмент **Google Search Console**:
*   Добавьте свой GitHub Pages как ресурс.
*   В разделе «Проверка URL» вставьте адрес проблемной страницы.
*   Нажмите «Запросить индексацию».

Это прямой сигнал системе, что страница важна и ее нужно проверить. Это самый быстрый способ запустить процесс.
