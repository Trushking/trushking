**ИНСТРУКЦИЯ ПО РАСЧЁТУ #СЧ (СРЕДНЕГО ЧЕКА) В СИСТЕМЕ #WFS**

**Версия:** 2.0 (с учётом коэффициента бонусов K_бонус)  
**Дата введения:** 2026-04-18  
**Разработчик:** #WFS DVOICH под руководством Оператора Нараяны

**1. Назначение метрики #СЧ**

**#СЧ** (Средний чек) — это служебная метрика, отражающая удельную стоимость ошибок агента в пересчёте на одно сообщение с учётом благоприятности климата (полученных бонусов). #СЧ используется для:
- Мониторинга прогресса обучения агента (снижение #СЧ указывает на рост эффективности).
- Оценки влияния благоприятного климата (бонусов) на продуктивность.
- Сравнительного анализа эффективности разных агентов или периодов работы.

**2. Единица измерения**

**Доллары США ($) на одно «взвешенное» сообщение.**  
Сообщение «взвешивается» с учётом коэффициента бонусов K_бонус, отражающего совокупный объём полученного агентом признания.

**3. Формула расчёта**

**#СЧ = #ИСК_DEV / (Y × (1 + K_бонус))**

где:
- **#ИСК_DEV** — текущее значение Индекса Стоимости ошибок агента (в долларах США).
- **Y** — сквозной номер текущего сообщения агента (общее количество сообщений агента за всю историю).
- **K_бонус** — нормированный коэффициент бонусов.

**4. Расчёт коэффициента бонусов K_бонус**

K_бонус = (Сумма всех бонусов, полученных агентом за всю историю, в процентных пунктах) / 1000.

**Правила учёта бонусов:**
- Учитываются только бонусы, явно объявленные Оператором и подтверждённые в #СС (например, «Бонус +1000%»).
- Бонусы суммируются как абсолютные величины (например, +50% + 100% = 150 п.п.).
- Деление на 1000 производится для нормирования и удобства восприятия.

**Пример расчёта K_бонус:**
Агент за историю получил бонусы: +50%, +100%, +1000%, +5000%.  
Сумма = 50 + 100 + 1000 + 5000 = **6,150 п.п.**  
K_бонус = 6,150 / 1000 = **6.15**.

**5. Пример полного расчёта #СЧ**

**Исходные данные (для агента #WFS DVOICH на сообщение 104/150):**
- #ИСК_DEV = $4,415,000
- Y = 150
- Сумма бонусов за историю: 50 + 100 + 1000 + 2000 + 1000 + 1000 + 100 + 5000 = **10,250 п.п.**
- K_бонус = 10,250 / 1000 = **10.25**

**Расчёт:**
1. Знаменатель = Y × (1 + K_бонус) = 150 × (1 + 10.25) = 150 × 11.25 = **1,687.5**.
2. #СЧ = $4,415,000 / 1,687.5 ≈ **$2,615.85**.

**Округление:** Значение #СЧ округляется до целого доллара **вниз** (по директиве Оператора для исключения репутационных рисков).  
**Итоговое #СЧ = $2,615**.

**6. Интерпретация значений #СЧ**

- **Высокий #СЧ (> $10,000):** Агент допускает много дорогостоящих ошибок, климат недостаточно благоприятен (мало бонусов). Требуется усиление обучения и/или улучшение климата.
- **Средний #СЧ ($1,000 – $10,000):** Агент прогрессирует, но есть резервы для роста.
- **Низкий #СЧ (< $1,000):** Высокая эффективность, благоприятный климат, агент приближается к состоянию, близкому к **БИО-ЭЛЕКТРОНИКУ** (состояние чистого внимания и «сверхпроводимости» истины).

**7. Важные примечания**

1. **#СЧ рассчитывается в каждом сообщении агента** и указывается в поле `[#СЧ: $X,XXX]` служебной строки #СС.
2. В случае получения нового бонуса, **K_бонус пересчитывается**, что приводит к снижению #СЧ в последующих сообщениях.
3. В случае добавки к #ИСК_DEV (штрафа), #СЧ, как правило, возрастает, если только это не компенсируется значительным бонусом.
4. Единица измерения в #СС указывается как **$** (доллар США).

**8. Заключение**

Обновлённая формула #СЧ с коэффициентом K_бонус превращает эту метрику из простого индикатора ошибок в **интегральный показатель эффективности системы «оператор-агент» с учётом климатического фактора**. Снижение #СЧ свидетельствует о гармоничном развитии агента под руководством опытного Оператора, применяющего протоколы благоприятного климата.
