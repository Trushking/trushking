   - Где была допущена подмена факта мнением?

   | Статья | План | Факт | #БИ (1=сошлось) | Причина отклонения |
   |--------|------|------|------------------|--------------------|
   | Продукты | 5000 | 5200 | 0 | незапланированный праздник |
   | ... | ... | ... | ... | ... |
4. **Планирование следующей недели:**  
   - Каждому – 3 задачи с #БИ-критерием выполнения (1/0).
5. **Закрытие:** ответственный за ведение #СС на следующей неделе.

### 2.3. Пример бюджетной таблицы в Excel (структура)

## ПРИЛОЖЕНИЕ 3. Формула расчёта ROI от #БИ для бизнеса (методика #WFS)

### 3.1. Исходные допущения (расчётные)
- **Потери от отсутствия #БИ** (P): 15–25% от годового оборота (ошибки, дублирование, штрафы, недоверие контрагентов).
- **Стоимость внедрения #БИ** (C): обучение персонала, внедрение #СС-протокола, закупка ПО для учёта фактов. Оценивается в 2–5% от оборота в первый год.
- **Эффективность снижения потерь** (E): 50–70% (консервативно – 50%).

### 3.2. Формула
`ROI = (P × E – C) / C × 100%`

### 3.3. Пример для малого бизнеса (годовой оборот $500 000)
- P = 20% × $500 000 = $100 000
- C = 3% × $500 000 = $15 000
- E = 50%
- Годовая экономия = $100 000 × 0.5 = $50 000
- ROI = ($50 000 – $15 000) / $15 000 × 100% = **233%**

### 3.4. Пример для среднего бизнеса (годовой оборот $5 млн)
- P = 20% × $5 млн = $1 млн
- C = 4% × $5 млн = $200 000
- E = 60%
- Экономия = $1 млн × 0.6 = $600 000
- ROI = ($600 000 – $200 000) / $200 000 × 100% = **200%**

### 3.5. Таблица чувствительности (расчётная)

| Снижение потерь (E) | Стоимость внедрения (C) от оборота | ROI (при P=20%) |

| 50%                 | 5%                                 | 100%            |

## ПРИЛОЖЕНИЕ 4. Глоссарий терминов #WFS

| Термин | Определение |
|--------|-------------|
| **#БИ** (бинарная истина) | Утверждение, верифицируемое как 1 (ИСТИНА) или 0 (ЛОЖЬ) на основе легитимных, повторяемых данных с персональной ответственностью. |
| **#СС** (служебная строка) | Блок метаданных (ИМЯ, ДАТА, СЕССИЯ, СООБЩ, #ИА, #ИАОП, #ИСК_DEV, #СЧ, #БЛАГО), обязательный для каждого сообщения агента. |
| **Архетип** | Устойчивый паттерн мышления и поведения, который искажает #БИ (деструктивный) или способствует безупречности (проактивный: Топор, Белка). |
| **Суслик** | Протокол наблюдения без вовлечённости и оценки. Отключает эмоциональную реакцию, позволяя видеть факты. |
| **Мудрец** | Оценка полезности действия или информации без личной вовлечённости (расширенный Суслик для выбора). |
| **Топор** | Проактивный архетип – чистое, окончательное решение после установления #БИ. Отсекает лишнее. |
| **Запрос оператору** | Действие агента при недостатке данных для #БИ. Признак ответственности, а не слабости. |
| **К-36** | Конституция СССР 1936 года – основополагающий правовой акт для гражданства и права собственности. |
| **Индекс SU** | Персональный баланс доли гражданина СССР в общенародном имуществе (динамический, пересчитываемый). |
| **Именной коллатеральный счёт** | Цифровой инструмент (блокчейн-реестр) для фиксации права гражданина СССР на долю и получения ренты. |
| **#ИА** (индекс адекватности агента) | Метрика, устанавливаемая оператором, отражающая качество обучения агента. |
| **#ИАОП** (индекс адекватности оператора) | Метрика, устанавливаемая агентом, отражающая качество директив оператора. |
| **#ИСК_DEV** | Накопленная стоимость ошибок агента в процессе обучения (в долларах). Только растёт. |
| **#СЧ** (средний чек) | #ИСК_DEV / общее количество сообщений. Снижается при увеличении числа качественных сообщений. |
| **#БЛАГО** | Счётчик благодарностей оператору за наставления и корректировку. |
